Bridge di pubblicazione

Il pulsante «Vai» invia testo e media (immagini/video in base64) al server locale (bridge/publish_server.py) che usa token memorizzati come variabili d’ambiente — non nel browser. Configura URL e chiave qui; copia la stessa chiave nella variabile PUBLISH_SECRET del bridge.

Intelligenza artificiale (generazione post)

«Genera con IA» chiama POST /generate sullo stesso bridge. Il modello viene eseguito dal bridge con la chiave qui sotto oppure con variabili d’ambiente (OPENAI_API_KEY, ANTHROPIC_API_KEY in plotino.env) affinché la chiave non resti solo nel browser.

Variabili sul bridge (riferimento)

PUBLISH_SECRET X_BEARER_TOKEN · OAuth 2 Bearer utente con scope tweet.write FACEBOOK_PAGE_ACCESS_TOKEN · FACEBOOK_PAGE_ID LINKEDIN_ACCESS_TOKEN · LINKEDIN_AUTHOR_URN (es. urn:li:person:…) INSTAGRAM_ACCESS_TOKEN · INSTAGRAM_ACCOUNT_ID (Meta Graph) TIKTOK_ACCESS_TOKEN (Content Posting API — beta) YOUTUBE_* — upload/resumable richiede OAuth Google Cloud / Data API (stub sul bridge) OPENAI_API_KEY · AI_MODEL — generazione IA (alternativa: chiave nel form sopra) ANTHROPIC_API_KEY — se scegli Claude nel form Provider OPENAI_BASE_URL — override endpoint compatibile OpenAI

Piattaforme da poter pubblicare

Abilita solo i canali per cui hai configurato il bridge. La selezione effettiva avviene nel concentratore.

Bridge e URL sono salvati in locale. La chiave IA nel form è opzionale; preferisci plotino.env sul bridge per non salvare segreti nel browser.
